# LiteLLM Helm Chart with Custom Values

Same chart as `litellm-chart`, but with a scaffolded values file so you can
see and edit every configurable option.

## How these files were created

```bash
composed init
composed add oci://docker.litellm.ai/berriai/litellm-helm
composed init --helm-values
composed up
```

`composed init --helm-values` runs `helm show values` for each helm service
and writes the raw output to `<name>.values.yaml`. The file is wired into
`composed.yaml` via `values_file:` so Helm picks it up at build time.

Edit the values file, then rebuild:

```bash
composed up
```

## Files

| File | Source |
|------|--------|
| `composed.yaml` | Generated by `composed init` + `composed add` + `composed init --helm-values` |
| `litellm-helm.values.yaml` | Generated by `composed init --helm-values` (identical to `helm show values`) |
| `docker-compose.yaml` | Generated by `composed up` (build step) |
