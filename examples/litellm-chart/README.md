# LiteLLM Helm Chart Example

A single Helm chart turned into a Docker Compose stack using `composed`.

## How these files were created

```bash
composed init
composed add oci://docker.litellm.ai/berriai/litellm-helm
composed up
```

Three commands. `composed add` probed the OCI registry, detected a Helm chart,
and wrote the service entry. `composed up` built the Compose file (pulling the
chart, rendering templates, translating K8s manifests) and started the stack.

## Files

| File | Source |
|------|--------|
| `composed.yaml` | Generated by `composed init` + `composed add` |
| `docker-compose.yaml` | Generated by `composed up` (build step) |

## Customising

Scaffold the full default values file:

```bash
composed init --helm-values
```

This creates `values-litellm-helm.yaml` with every configurable option
(comments included) and wires it into `composed.yaml` via `values_file:`.
Edit the file, then rebuild:

```bash
composed up
```

Or use inline overrides at add time:

```bash
composed add oci://docker.litellm.ai/berriai/litellm-helm \
  --set image.tag=main-stable \
  --set masterkey=sk-1234 \
  --set db.deployStandalone=true
```
